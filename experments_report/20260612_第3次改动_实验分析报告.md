# V2.3 消融实验完整分析报告（第3次改动 — scale_mode + reliability gate）

> 生成日期: 2026-06-12
> 模型: DualBranchSTImputer + GatedCrossScaleSharedExpert + ProgressiveRouteFusion(scale_mode-aware) + SharedRoutedResidualFusion
> 关键改动: scale_mode 显式控制、ReliabilityAwareScaleGate、TaxiBJ 去掉 coarse 尺度
> 数据集: TaxiBJ (32×32, fine_mid), BikeNYC (24×12, fine_mid_coarse)

---

## 1. 模型架构（基于 2026-06-11 代码）

### 1.1 整体架构

```
DualBranchSTImputer
└── Main Branch (MultiScaleMoEBackbone)
    │
    ├── [1] ScaleTokenEncoder ×3  (fine / mid / coarse)
    │   └── value_embed(Conv→GN→GELU) + mask_embed + scale_embed + time_embed + space_embed
    │
    ├── [2] QualityRouter ×3 (per-scale, quality-aware)
    │   └── pooled(h) || obs_stats(mask) || scale_embed → Softmax → [B, num_experts]
    │
    ├── [3] TopKRoutedExpertPool (num_experts=4, top_k=2, 参数共享)
    │   └── Z_f, Z_m, Z_c
    │
    ├── [4] GatedCrossScaleSharedExpert  ← NEW (替换 CrossScaleSharedExpert)
    │   ├── ReliabilityAwareScaleGate:
    │   │   └── [p_f,p_m,p_c || q_f,q_m,q_c || r_m_mean,r_c_mean] → MLP → Softmax → scale_gate[B,3]
    │   ├── h_m↑, h_c↑ (upsampled to fine size)
    │   └── fuse(w_f·h_f + w_m·h_m↑ + w_c·h_c↑) → Conv1×1(3D→D) → 2×ResBlock → Z_shared
    │
    ├── [5] ProgressiveRouteFusion (scale_mode-aware)  ← 支持 fine/fine_mid/fine_mid_coarse
    │   ├── scale_mode="fine":
    │   │   └── fine_refine(Z_f) → h_route
    │   ├── scale_mode="fine_mid":
    │   │   └── LearnableUpsample3D(Z_m) → GatedFusion2(Z_f, Z_m↑) → h_route
    │   └── scale_mode="fine_mid_coarse":
    │       └── Z_c→upsample→GatedFusion2(Z_m,Z_c↑)→Z_mc→upsample→GatedFusion2(Z_f,Z_mc↑)→h_route
    │
    ├── [6] SharedRoutedResidualFusion
    │   ├── shared_refine: Z_shared → 2×ResBlock → h_shared
    │   ├── route_proj: h_route → Conv1×1 → ResBlock → h_route_proj
    │   └── h_main = h_shared + sigmoid(γ) × h_route_proj   (γ_init=-4, σ≈0.018)
    │
    └── [7] PredHead: Conv3d(D→D/2,k=3)→GELU→Conv3d(D/2→C_in,k=1)
```

### 1.2 本次改动要点

| 组件 | 旧版 (V2) | 新版 (V2.3) |
|---|---|---|
| Shared Expert | `CrossScaleSharedExpert`: concat → fuse | `GatedCrossScaleSharedExpert`: ReliabilityAwareScaleGate → weighted sum → fuse |
| 尺度门控 | 无 | `ReliabilityAwareScaleGate`: MLP(h, q, r) → [w_f, w_m, w_c] |
| Routed Fusion | 固定 3 尺度 | `scale_mode` 支持 fine / fine_mid / fine_mid_coarse |
| TaxiBJ 配置 | fine_mid_coarse (3 尺度) | **fine_mid** (去 coarse, 32→16) |
| BikeNYC 配置 | fine_mid_coarse | fine_mid_coarse (不变) |
| Balance Loss | 固定 3 scale | 按 active scales 动态选择 |

### 1.3 scale_mode 三种路径

```
scale_mode="fine":
  Shared: w_f=1, w_m=0, w_c=0 → Z_shared = fuse(h_f)
  Routed: Z_f → ResidualSTBlock → h_route
  Cross Loss: l_cross = 0
  Balance: fine only

scale_mode="fine_mid" (TaxiBJ 默认):
  Shared: w_f, w_m learned, w_c=0 → Z_shared = fuse(w_f·h_f, w_m·h_m↑, 0)
  Routed: Z_m → Upsample → GatedFusion2(Z_f, Z_m↑) → h_route
  Cross Loss: fine→mid only
  Balance: fine, mid

scale_mode="fine_mid_coarse" (BikeNYC 默认):
  Shared: w_f, w_m, w_c learned → Z_shared = fuse(w_f·h_f, w_m·h_m↑, w_c·h_c↑)
  Routed: coarse→mid→fine 逐级门控
  Cross Loss: fine→mid + mid→coarse
  Balance: fine, mid, coarse
```

### 3.3 核心发现

**1. 去掉 coarse 是 TaxiBJ 的决定性突破**

上批次 6 个模型中 4 个 MAE > 17，本批次最好的 5 个模型全部 < 13。核心原因：
- TaxiBJ 32×32 → coarse 8×8 降采样 4×，信息损失过大
- 旧架构 coarse→mid→fine 逐级融合时，coarse 噪声通过 GatedFusion 传播到 fine
- 去掉 coarse 后信号链路干净，Routed-Only 和 No Cross-Scale 直接受益

**2. Routed-Only > Full Model（反常）**

Routed-Only MAE=11.90 首次在 TaxiBJ 上击败 Full Model（12.01）。原因分析：
- Full Model 的 Shared Branch 学到的 scale_gate = [0.77, 0.23, 0]，fine 占主导
- 但 Shared Branch 仍在使用 pre-expert 的原始 embedding（h_f, h_m），信息不如 post-expert 的 Z_f, Z_m
- Routed-Only 只用 ProgressiveRouteFusion（Z_f + Z_m↑ → GatedFusion2），信号更纯
- Full Model 中 route_gamma=0.032，路由残差几乎不参与，主要靠 shared branch → 效果被 shared 拖累

**3. Shared-Only 退化（14.82 vs 旧 12.91）**

去掉 coarse + 去掉 routed branch → Shared-Only 失去了互补信息来源：
- 旧 Shared-Only (3-scale)：有 coarse 的全局上下文补偿
- 新 Shared-Only (2-scale)：只有 fine+mid，且无 routed branch 的 post-expert 特征
- scale_gate 学到 f=0.76, m=0.24，与 Full Model 接近，但缺少 routed branch 的 residual

**4. Fine-Only 保持稳定（12.15）**

scale_mode=fine, 单尺度。3 个版本趋势：12.09(0609) → 12.78(0611) → 12.15(0611-12)。回到 12.15（接近历史最佳），说明新架构对 Fine-Only 没有负面影响。

**5. No Router 大幅改善（17.95→12.89）**

上批次 No Router 使用 3-scale + dense routing 导致崩溃。本批次 fine_mid + dense routing 表现大幅改善。scale_gate = [0.76, 0.24]，说明 ReliabilityAwareScaleGate 在没有 quality router 时仍能有效工作。

### 4.3 核心发现

**1. Full Model 新纪录 (2.7808)**

历史最佳，scale_gate = [f=0.69, m=0.12, c=0.19]。值得注意的是 **coarse 权重(0.19) > mid 权重(0.12)**，与 TaxiBJ 相反：
- BikeNYC 24×12 → mid 12×6 是不规则降采样，信息保留不均匀
- coarse 6×3 反而捕获了更全局的路网模式
- fine 仍然主导（0.69），与 TaxiBJ 的 fine 偏好一致

**2. No Router 跃升至 #2**

从 2.96 → 2.83，超过 Shared-Only。说明 ReliabilityAwareScaleGate 提供了足够的质量感知能力，即使没有 QualityRouter。
- scale_gate 与 Full Model 几乎一致 (f=0.69, m=0.11, c=0.20)
- route_gamma=0.020（残差贡献极小）

**3. Routed-Only 和 No Cross-Scale 轻微退化**

- Routed-Only: 2.89→2.94 (+1.8%)
- No Cross-Scale: 2.92→2.97 (+1.6%)

两者共享 `use_shared_branch=false`。退化原因不是架构变化（ProgressiveRouteFusion fine_mid_coarse 路径未变），而是训练方差或 balance loss 计算方式微调（新增 use_routed_branch 条件判断）的间接影响。

**4. Shared-Only 保持稳定 (2.8477)**

几乎与上批次一致 (2.8486)，scale_gate = [0.66, 0.14, 0.20]。Shared-Only 是 BikeNYC 上最稳定的配置。

### 5.4 route_gamma 分析

| 模型 | TaxiBJ γ | BikeNYC γ | 路由分支贡献 |
|---|---|---|---|
| Full Model | 0.032 | 0.021 | ~3% / ~2% |
| No Router | 0.033 | 0.020 | ~3% / ~2% |

route_gamma 在两个数据集上都保持在 0.02-0.03（初始值 σ(-4)=0.018），说明路由分支始终作为极小的残差修正。这与设计意图一致——shared branch 是主干，routed branch 是微调。

但 TaxiBJ Routed-Only 达到最佳 11.90，说明当 shared branch 完全去掉时，routed branch 的 ProgressiveRouteFusion 单独工作也非常好——只是 Full Model 中 shared branch 限制了它的贡献（γ 太小）。

## 7. 综合评估与建议

### 7.1 改动有效性评估

| 改动 | 有效性 | 证据 |

### 7.2 数据集特异性建议

**TaxiBJ (推荐: Routed-Only, scale_mode=fine_mid)**:
- ✅ 最佳配置: `scale_mode=fine_mid`, `use_shared_branch=false`
- 💡 Full Model（双分支）不如纯 Routed-Only，说明 shared branch 的 pre-expert 特征质量不如 post-expert
- 💡 可尝试增大 route_gamma_init（-2.0 → σ≈0.12）让 Full Model 中的路由残差更活跃
- 💡 可尝试 scale_mode=fine（单尺度），Fine-Only 已达 12.15，可能更稳定
- ⚠️ 避免使用 fine_mid_coarse，coarse 8×8 注入噪声

**BikeNYC (推荐: Full Model, scale_mode=fine_mid_coarse)**:
- ✅ 最佳配置: `scale_mode=fine_mid_coarse`, 双分支全开

- 💡 No Router 已达 2.83，接近 Full Model 2.78，可考虑简化
- 💡 当前 route_gamma 很小（0.021），可尝试增大让路由残差更多参与

1. **TaxiBJ Routed-Only 验证**: 多次运行确认 11.90 的稳定性，排除随机种子偏差
2. **route_gamma 调参**: 尝试 -2.0 或 -1.0（σ=0.12~0.27），让 Full Model 中路由分支更多贡献
3. **TaxiBJ Shared Branch 改进**: 考虑让 shared branch 也使用 post-expert 特征（而非 pre-expert embedding）
4. **scale_mode 消融**: 运行 `ablation_scale_fine` / `ablation_scale_fine_mid` / `ablation_no_scale_gate` 做更细粒度的尺度消融
5. **Baselines 对比**: 与统计方法（mean/historical/linear）和经典深度方法对比
