**在 rate=0.4 附近发生相变：**

- **Rate ≤ 0.2**：两分支贡献接近，Shared-Only 略优（路由多余）
- **Rate = 0.4**：Routed-Only 最优，路由分支找到其"甜区"——足够多的数据让 gate 有效，足够多的缺失让路由有意义
- **Rate ≥ 0.6**：路由分支变成负资产，Shared-Only 和 No-Router 双双超越 Full Model

## 5. 逐消融深度分析

### 5.1 Shared-Only — 🟢 最佳模型

移除路由分支后，在 **6/8 场景下最优**。唯一表现差的是 rate=0.4（fixed 排第6, random 排第5）。

**关键洞察**：共享分支是整个架构的"安全网"。路由分支只在特定条件下（rate≈0.4）有明确的正贡献，其他时候可有可无甚至有害。

### 5.2 Routed-Only — 🟡 rate=0.4 的王者

| Rate | Fixed Δ | Random Δ |
|------|:-------:|:--------:|
| 0.2 | +0.014 | −0.002 |
| 0.4 | **−0.003** | **−0.022** |
| 0.6 | +0.035 | +0.033 |
| 0.8 | +0.058 | +0.044 |

路由分支在 rate=0.4 达到最佳表现，之后随缺失率急剧退化。缺失率从 0.4→0.6，路由分支退化幅度是所有组件中最大的（~0.04-0.05 MAE）。

### 5.3 No-Router — 🟡 高缺失率下的低成本替代

移除动态路由（保持双分支但固定混合权重）在 rate≥0.6 时优于 Full Model。说明问题不在双分支结构本身，而在**动态路由的决策质量**。

### 5.4 Fine-Only — 🔴 多尺度不可或缺

缺失率 0.2→0.8 的退化幅度最大：
- Fixed: Δ从 +0.017 → +0.075（退化 **+0.058**）
- Random: Δ从 +0.014 → +0.056（退化 **+0.042**）

**缺失越多，多尺度越重要。** 高缺失率下粗/中尺度提供了不可或缺的上下文信息。

### 5.5 Fixed-Scale Experts — 🔴 压倒性最差

8/8 场景全部排名第 6 或第 7，无一例外。可学习的尺度门控是整个架构中最重要的单一组件。

## 6. Fixed vs Random 掩码模式

两种缺失模式的结果高度一致（Full Model MAE 差异 < 0.02），模型对缺失的空间结构不敏感。

## 7. 总结

### 主要结论

1. **Full Model 从来不是最优。** 双分支设计存在内在矛盾——一个分支总在拖累另一个。

2. **缺失率 0.4→0.6 是相变边界。** 低于 0.6，路由分支有正或中性贡献；高于 0.6，路由变成明确的负资产。

3. **Shared-Only 是最稳健的架构。** 简单、高效，在绝大多数场景下最优。路由分支仅在 rate≈0.4 的"甜区"提供了额外的价值。

4. **可学习尺度门控 + 多尺度架构是不可妥协的基础。** Fixed-Scale Experts 全部垫底，Fine-Only 高缺失率下严重退化。

### 设计建议

当前模型的动态路由在高缺失率下不可靠。你之前做的 **reliability-aware scale gate** 方向正确——需要一种机制，在数据稀疏时自动抑制路由分支的贡献权重，让模型退化到接近 Shared-Only 的行为。理想效果：

```
可靠性高 (rate≤0.4) → route_alpha ≈ 0.5（两分支等权）
可靠性低 (rate≥0.6) → route_alpha → 0（退化为 Shared-Only）
```
