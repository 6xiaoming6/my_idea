## 1. 实验设计

### 1.1 模型架构（当前代码实际架构 — ProgressiveScaleGatedFusion）

```
DualBranchSTImputer
├── Main Branch (MultiScaleMoEBackbone)
│   ├── ScaleTokenEncoder ×3    (fine:32×32, mid:16×16, coarse:8×8)
│   ├── QualityRouter ×3        (观测质量感知的动态路由)
│   ├── Intra-Scale Routed Branch
│   │   └── TopKRoutedExpertPool (top-k=2 稀疏专家池)
│   │       → Z_f [B,D,T,32,32]
│   │       → Z_m [B,D,T,16,16]
│   │       → Z_c [B,D,T,8,8]
│   ├── Cross-Scale Shared Branch
│   │   └── CrossScaleSharedExpert (upsample+concat+conv)
│   │       → Z_shared [B,D,T,32,32]
│   ├── ProgressiveScaleGatedFusion   ← 按 design_v2-2(MOE融合改进) 实现
│   │   ├── Stage 1: coarse → mid
│   │   │   ├── LearnableUpsample3D (Z_c, 8×8→16×16)
│   │   │   └── GatedFusion2 (Z_m + Z_c_up) → Z_mc [B,D,T,16,16]
│   │   │       └── Conv3d(2D→2,k=1) → Softmax → 加权求和 → ResBlock
│   │   └── Stage 2: mid → fine
│   │       ├── LearnableUpsample3D (Z_mc, 16×16→32×32)
│   │       └── GatedFusion3 (Z_f + Z_mc_up + Z_shared) → h_main [B,D,T,32,32]
│   │           └── Conv3d(3D→3,k=1) → Softmax → 加权求和 → ResBlock
│   └── PredictionHead (Conv3D → GELU → Conv3D) → x_hat_main
└── Aux Branch (NullResidualBranch, 当前禁用)
```

### 1.2 架构变更说明

本次实验所用代码已按 `design_v2-2(MOE融合模块改进).md` 实现 ProgressiveScaleGatedFusion，
替换了旧版的简单 concat fusion。核心改进：

| 组件 | 旧版 (原报告) | 新版 (本次) |
|---|---|---|
| Fusion 模块 | `Concat([Z_f,Z_m_up,Z_c_up,Z_shared]) → Conv1×1 → 2×ResBlock` | `ProgressiveScaleGatedFusion` (逐级门控融合) |
| 上采样 | `F.interpolate` 直接 8×8→32×32 | `LearnableUpsample3D`: interpolate + Conv3D + GroupNorm + GELU + ResBlock |
| 融合逻辑 | 隐式卷积学习 | 显式 softmax 门控: gate_16 (coarse/mid 权重), gate_32 (fine/mc/shared 权重) |
| 尺度关系 | 并行 concat (flat) | 逐级 coarse→mid→fine (hierarchical) |
| 对应设计文档 | design_v2-2.md | design_v2-2(MOE融合模块改进).md |

> 所有消融实验均使用 ProgressiveScaleGatedFusion。当某分支被关闭时（如 routed_br=False），对应输入为 zeros，门控机制自动学到忽略该分支。
