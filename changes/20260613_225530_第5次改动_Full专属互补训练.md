## 3. 代码层面的详细改动

### 3.1 `fusion.py`

`ExpertEnhancedSharedInput` 的默认输入模式从 `hybrid` 改为 `pre`。

旧逻辑默认会把 routed expert 输出注入 shared branch 输入中：

```text
h_shared_input = h_pre + beta * z_routed
```

这种方式在 V2.4 实验中容易让 routed 分支的噪声提前污染 shared 主干。新默认逻辑改成：

```text
h_shared_input = h_pre
```

也就是 shared branch 默认只看原始多尺度 embedding，`hybrid`、`post`、`concat_hz` 等增强模式仍保留在代码中，可作为后续消融实验使用。

`SharedRoutedResidualFusion` 新增 `route_dropout` 参数。routed branch 经过 `project_route` 投影之后，会先进入 dropout，再参与融合：

```text
h_route_proj = project_route(h_route)
h_route_proj = route_dropout(h_route_proj)
h_main = h_shared + sigmoid(route_gamma) * h_route_proj
```

这让 routed 分支在 Full Model 中提供残差修正，同时降低 routed 特征过拟合或过度污染 shared 主干的风险。

### 3.2 `main_branch.py`

`MultiScaleMoEBackbone` 的默认结构参数按 V2.5 调整：

- `branch_fusion_mode` 默认从 `shared_plus_routed_residual`/配置中的 `adaptive_gate` 调整为 `residual`
- `route_gamma_init` 默认从 `-4.0` 调整为 `-3.0`
- `shared_input_mode` 默认从 `hybrid` 调整为 `pre`
- 新增 `route_dropout`
- 新增 `enable_branch_aux`
- 新增 `enable_complementary_loss`

模型新增两个 Full 专属辅助预测头：

```text
shared_aux_head: H_shared -> x_hat_shared
route_aux_head:  H_route_proj -> x_hat_route
```

两个辅助头的结构和主预测头保持一致，都是：

```text
Conv3d(dim, dim // 2, kernel_size=3, padding=1)
GELU
Conv3d(dim // 2, c_in, kernel_size=1)
```

forward 输出新增：

- `x_hat_shared`
- `x_hat_route`
- `enable_branch_aux`
- `enable_complementary_loss`

这两个辅助预测只在完整双分支结构中启用：

```text
is_full = use_shared_branch and use_routed_branch

if is_full and enable_branch_aux:
    x_hat_shared = shared_aux_head(h_shared)
    x_hat_route = route_aux_head(h_route_proj)
else:
    x_hat_shared = None
    x_hat_route = None
```

因此 Shared-Only、Routed-Only、Fine-Only、No Router 等消融配置不会拿到 Full Model 的辅助重建优势。

### 3.3 `losses.py`

新增 `complementary_loss(h_shared, h_route)`，用于约束 shared branch 和 routed branch 的特征不要学成完全重复的信息：

```text
hs = normalize(flatten(H_shared))
hr = normalize(flatten(H_route))
L_comp = mean(cosine_similarity(hs, hr)^2)
```

`compute_main_stage_loss` 新增三个 loss 项：

- `l_shared_aux`：`x_hat_shared` 对缺失位置的辅助重建损失
- `l_route_aux`：`x_hat_route` 对缺失位置的辅助重建损失
- `l_complementary`：`H_shared` 与 `H_route_proj` 的互补性约束

新的 Full loss 形式为：

```text
L =
  L_main
+ lambda_cross * L_cross
+ lambda_importance_balance * L_importance
+ lambda_load_balance * L_load
+ lambda_shared_aux * L_shared_aux
+ lambda_route_aux * L_route_aux
+ lambda_complementary * L_complementary
+ 其他已有可选项
```

其中 `l_shared_aux`、`l_route_aux`、`l_complementary` 都会写入训练日志。它们只有在 `use_shared_branch=true` 且 `use_routed_branch=true` 时才实际产生非零训练作用，否则保持为 0。

### 3.4 `engine.py`

优化器参数组默认倍率按 V2.5 降低：

- `gate_lr_mult`: `2.0 -> 1.0`
- `scalar_lr_mult`: `5.0 -> 2.0`

这样可以避免 gate 和标量融合参数在训练早期更新过猛，降低 Full Model 融合不稳定的风险。

训练和验证日志新增两个辅助预测头的指标统计。如果输出中存在 `x_hat_shared` 或 `x_hat_route`，会额外记录：

- `mae_shared_aux`
- `rmse_shared_aux`
- `mape_shared_aux`
- `mae_route_aux`
- `rmse_route_aux`
- `mape_route_aux`

诊断日志中新增 `route_alpha`，其值等于 `sigmoid(route_gamma)`，便于直接观察 routed residual 在 Full 输出中的实际强度。

## 4. 模型层面的详细改动

### 4.1 旧的 V2.4 默认问题

V2.4 默认路径更激进：

```text
Shared 输入 = pre embedding + routed expert feature
Branch 融合 = AdaptiveBranchGate(sample-level)
训练策略 = 更高 gate/scalar 学习率 + warmup_cosine + early stopping
```

实验中出现两个问题：

- `hybrid` shared 输入会把 routed feature 的噪声提前注入 shared 主干，导致 shared branch 退化。
- `AdaptiveBranchGate` 容易学成全局 shared 偏置，让 route branch 几乎没有贡献。

### 4.2 新的 V2.5 默认结构

V2.5 不继续堆新模块，而是把 Full Model 固定为更清晰的双分支互补结构：

```text
输入:
  x_f, x_m, x_c, mask, time/query feature

多尺度编码:
  x_f -> H_f
  x_m -> H_m
  x_c -> H_c

Shared Branch:
  H_f, H_m, H_c
    -> reliability-aware scale gate
    -> GatedCrossScaleSharedExpert
    -> H_shared

Routed Branch:
  H_f, H_m, H_c
    -> QualityRouter
    -> TopK shared expert pool
    -> ProgressiveRouteFusion
    -> H_route
    -> route projection
    -> route dropout
    -> H_route_proj

Full Residual Fusion:
  alpha = sigmoid(route_gamma)
  H_main = H_shared + alpha * H_route_proj

预测:
  x_hat_main   = pred_head(H_main)
  x_hat_shared = shared_aux_head(H_shared)      仅 Full 训练启用
  x_hat_route  = route_aux_head(H_route_proj)   仅 Full 训练启用

最终输出:
  x_hat_final = coarse correction / output wrapper 后的最终结果
```

### 4.3 关键张量形状

以 smoke 测试配置为例：

```text
x_hat_main:   [B, C, T, H, W]
x_hat_final:  [B, C, T, H, W]
x_hat_shared: [B, C, T, H, W]，Full 且 enable_branch_aux=true 时存在
x_hat_route:  [B, C, T, H, W]，Full 且 enable_branch_aux=true 时存在
H_shared:     [B, D, T, H, W]
H_route_proj: [B, D, T, H, W]
```

其中 smoke 测试实际输出为：

```text
x_hat_main:   (2, 2, 6, 16, 16)
x_hat_shared: (2, 2, 6, 16, 16)
x_hat_route:  (2, 2, 6, 16, 16)
```

### 4.4 Full Model 和消融模型的区别

Full Model 同时拥有：

- shared branch
- routed branch
- residual route fusion
- shared auxiliary reconstruction
- route auxiliary reconstruction
- complementary loss
- cross-scale loss
- router balance loss

消融模型会按配置关闭对应结构，并额外关闭 Full 专属训练项。例如：

- Fine-Only：关闭 shared branch，并关闭 branch aux / complementary loss
- Shared-Only：关闭 routed branch，并关闭 route aux / complementary loss
- Routed-Only：关闭 shared branch，并关闭 shared aux / complementary loss
- No Router：关闭 router，并关闭 complementary loss
- No Cross-Scale：关闭 shared branch，并关闭 shared aux / complementary loss

这样消融实验不再共享 Full 的双分支互补监督优势，更符合“去掉关键模块后性能应下降”的实验目标。

```json

  "enable_branch_aux": false,
```

## 6. 改动后的模型架构总结

本次改动后的 Full Model 可以概括为：

```text
                ┌──────────────────────────────┐
                │ Input Multi-Scale Sequences  │
                │ x_f, x_m, x_c, masks, q      │
                └──────────────┬───────────────┘
                               │
                 ScaleTokenEncoder per scale
                               │
             ┌─────────────────┴─────────────────┐
             │                                   │
     Shared Branch                       Routed Branch
     pre embeddings                      QualityRouter
     reliability scale gate              TopK shared experts
     cross-scale shared expert           ProgressiveRouteFusion
             │                                   │
        H_shared                         H_route_proj
             │                                   │
             └──────────────┬────────────────────┘
                            │
                 H_main = H_shared + alpha * H_route_proj
                 alpha = sigmoid(route_gamma)
                            │
                       pred_head
                            │
                       x_hat_main
                            │
                     final output path
                            │
                       x_hat_final

     Full-only auxiliary heads:
        H_shared     -> shared_aux_head -> x_hat_shared
        H_route_proj -> route_aux_head  -> x_hat_route

     Full-only auxiliary losses:
        L_shared_aux, L_route_aux, L_complementary
```

核心思想是：shared branch 做稳定主干，routed branch 只以小比例 residual 的方式补充局部和缺失模式相关的信息，同时用辅助重建和互补约束保证 routed 分支不会被主 loss 忽略。

## 8. 兼容性和注意事项

- `shared_input_mode="hybrid"`、`post`、`concat_hz` 仍保留，可以作为后续消融使用，但不再是默认 Full 主路径。
- `branch_fusion_mode="adaptive_gate"` 仍保留，可以作为消融使用，但默认 Full 改为 `residual`。
- residual 模式下日志里的 `branch_gate` 是诊断用的 `[shared=1, route=alpha]`，不是 softmax 归一化概率。
- `enable_branch_aux` 和 `enable_complementary_loss` 建议只在 Full Model 开启。消融配置如果打开这些项，会破坏 V2.5 的实验设计假设。
- 本次只完成代码结构、配置和 smoke 级验证，最终是否达到 “Full Model 最优” 仍需要重新跑 TaxiBJ 和 BikeNYC 的完整实验确认。
