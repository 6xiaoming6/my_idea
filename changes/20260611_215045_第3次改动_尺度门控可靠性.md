# 尺度门控可靠性

## 修改时间

2026-06-11 21:50:45

## 修改依据

本次修改根据 `model_designs/design_2_4.md` 实现 v2.3 方案中的第一版 `scale_mode + reliability-aware scale gate`。

上一版模型已经是：

```text
Shared Branch 主干 + gamma * Routed Branch 残差
```

但它仍默认使用固定的三尺度结构：

```text
fine + mid + coarse
```

最新设计文档指出，TaxiBJ 上 `32 -> 16 -> 8` 的 coarse 尺度可能过粗，三尺度 Full Model 可能引入噪声；BikeNYC 上多尺度更有效。因此本次修改目标是：

1. 支持显式 `scale_mode`，让模型可以运行 `fine`、`fine_mid`、`fine_mid_coarse` 三种尺度组合。
2. 在 shared branch 中加入样本级 `ReliabilityAwareScaleGate`，让模型根据观测统计和 reliability map 给 fine/mid/coarse 分配权重。
3. 数据构造阶段保留 `r_m/r_c` reliability map。
4. loss、日志和配置同步理解 `scale_mode`。

## 代码层面的改动

### 1. 数据侧新增 reliability map

修改 `src/stmoe_imputer/data/transforms.py`。

`masked_pool2d_spatial()` 新增参数：

```python
return_reliability: bool = False
```

默认仍返回：

```python
x_down, m_down
```

显式开启后返回：

```python
x_down, m_down, r_down
```

其中：

```text
r_down = pooling window 内 observed cell 数量 / pooling window 面积
```

例如 `2x2` pooling：

```text
r = 0.00 -> 没有任何观测
r = 0.25 -> 1/4 观测
r = 1.00 -> 4/4 全观测
```

`ensure_multiscale()` 现在会写入：

```python
sample["r_m"]
sample["r_c"]
```

如果外部数据已经提供 `x_m_obs/m_m` 或 `x_c_obs/m_c`，但没有 reliability，则退化为：

```python
r_m = m_m.float()
r_c = m_c.float()
```

这保证旧数据格式也能运行。

### 2. NPZ 数据读取支持 `r_m/r_c`

修改 `src/stmoe_imputer/data/npz_dataset.py`。

如果 NPZ 文件中包含：

```text
r_m
r_c
```

会读取为 `[C=1,T,H,W]` 格式并放入 sample。若不包含，则仍由 `ensure_multiscale()` 使用 mask 近似补齐。

### 3. 新增 scale_mode 工具函数

新增 `src/stmoe_imputer/models/scale_utils.py`：

```python
get_active_scales(scale_mode)
is_scale_active(scale_mode, scale)
build_scale_active_mask(scale_mode, batch_size, device)
```

支持三种模式：

```text
fine             -> [fine]
fine_mid         -> [fine, mid]
fine_mid_coarse  -> [fine, mid, coarse]
```

`build_scale_active_mask()` 返回 `[B,3]` bool mask，对应 `[fine, mid, coarse]`。它用于 scale gate 中屏蔽 inactive scale。

### 4. 新增 ReliabilityAwareScaleGate

修改 `src/stmoe_imputer/models/fusion.py`，新增 `ReliabilityAwareScaleGate`。

输入包括：

```text
h_f, h_m, h_c: 多尺度 embedding 特征
q_f, q_m, q_c: observation statistics
r_m, r_c: reliability map
active_mask: [B,3]，由 scale_mode 决定
```

内部先做样本级池化：

```text
p_f = mean(h_f over T,H,W)
p_m = mean(h_m over T,H,W)
p_c = mean(h_c over T,H,W)
r_m_mean = mean(r_m)
r_c_mean = mean(r_c)
```

然后拼接：

```text
[p_f, p_m, p_c, q_f, q_m, q_c, r_m_mean, r_c_mean]
```

经过 MLP 输出：

```text
scale_gate: [B,3]
```

inactive scale 的 logits 会被 mask 到极小值，因此：

```text
scale_mode=fine      -> scale_gate = [1,0,0]
scale_mode=fine_mid  -> scale_gate 第 3 维 coarse 恒为 0
```

### 5. 新增 GatedCrossScaleSharedExpert

修改 `src/stmoe_imputer/models/fusion.py`，新增 `GatedCrossScaleSharedExpert`，并在主分支中替换原来的 `CrossScaleSharedExpert`。

新 shared branch 逻辑：

```text
h_m_up = upsample(h_m -> fine size)
h_c_up = upsample(h_c -> fine size)

scale_gate = ReliabilityAwareScaleGate(...)

h_f_w = w_f * h_f
h_m_w = w_m * h_m_up
h_c_w = w_c * h_c_up

concat(h_f_w, h_m_w, h_c_w)
  -> Conv3d(3D, D)
  -> ResidualSTBlock
  -> ResidualSTBlock
  -> z_shared
```

如果 `use_scale_gate=false`，则不经过 MLP，而是对 active scales 做均匀分配：

```text
fine_mid_coarse -> [1/3, 1/3, 1/3]
fine_mid        -> [1/2, 1/2, 0]
fine            -> [1, 0, 0]
```

### 6. ProgressiveRouteFusion 支持 scale_mode 分支

修改 `ProgressiveRouteFusion.forward()`：

```python
forward(z_f, z_m=None, z_c=None, scale_mode="fine_mid_coarse")
```

三种路径如下。

#### `scale_mode=fine`

```text
z_f -> fine_refine -> h_route
```

不执行 mid/coarse 融合。

#### `scale_mode=fine_mid`

```text
z_m -> LearnableUpsample3D -> z_m_to_f
z_f, z_m_to_f -> GatedFusion2 -> h_route
```

不执行 coarse -> mid。

#### `scale_mode=fine_mid_coarse`

保持上一版逐级路由融合：

```text
z_c -> up_c_to_m -> z_c_to_m
z_m, z_c_to_m -> GatedFusion2 -> z_mc
z_mc -> up_mc_to_f -> z_mc_to_f
z_f, z_mc_to_f -> GatedFusion2 -> h_route
```

输出字典新增：

```python
z_m_to_f
```

并保留：

```python
h_route
z_c_to_m
z_mc
z_mc_to_f
gate_16
gate_32_route
```

inactive gate 使用零张量占位，保证日志和下游代码 key 稳定。

### 7. MultiScaleMoEBackbone 接入 scale_mode 和 reliability

修改 `src/stmoe_imputer/models/main_branch.py`。

新增初始化参数：

```python
scale_mode="fine_mid_coarse"
use_scale_gate=True
use_reliability_gate=True
```

如果 `use_multiscale=false`，内部会强制：

```python
scale_mode = "fine"
```

forward 新增输入：

```python
r_m=None
r_c=None
```

如果没有传入，则用：

```python
r_m = m_m.float()
r_c = m_c.float()
```

主分支现在会构建：

```python
active_mask = build_scale_active_mask(self.scale_mode, batch_size, x_f.device)
```

并传给 shared branch：

```python
z_shared, h_m_up, h_c_up, scale_gate = self.cross_scale_shared_expert(
    h_f=h_f,
    h_m=h_m,
    h_c=h_c,
    q_f=q_f,
    q_m=q_m,
    q_c=q_c,
    r_m=r_m if use_reliability_gate else None,
    r_c=r_c if use_reliability_gate else None,
    active_mask=active_mask,
)
```

routed branch 调用：

```python
route_outputs = self.route_fusion(
    z_f=z_f,
    z_m=z_m,
    z_c=z_c,
    scale_mode=self.scale_mode,
)
```

输出新增：

```python
outputs["gates"]["scale_gate"]
outputs["scale_mode"]
outputs["use_scale_gate"]
outputs["use_reliability_gate"]
outputs["features"]["z_m_to_f"]
```

### 8. DualBranchSTImputer 传递 reliability

修改 `src/stmoe_imputer/models/imputer.py`。

调用主分支时新增：

```python
r_m=batch.get("r_m")
r_c=batch.get("r_c")
```

这样 synthetic dataset、NPZ dataset 以及旧数据都能兼容。

### 9. cross-scale loss 按 scale_mode 计算

修改 `src/stmoe_imputer/losses.py`。

`cross_scale_loss()` 新增参数：

```python
scale_mode="fine_mid_coarse"
```

逻辑变为：

```text
fine:
  l_cross = 0

fine_mid:
  l_cross = fine -> mid consistency

fine_mid_coarse:
  l_cross = fine -> mid consistency + mid -> coarse consistency
```

这避免 `scale_mode=fine` 时仍用 mid/coarse 监督模型，也避免 `fine_mid` 时被 coarse loss 干扰。

### 10. balance loss 按 active scale 统计

同样修改 `src/stmoe_imputer/losses.py`。

`moe_balance_loss()` 新增：

```python
scale_names=("fine", "mid", "coarse")
```

`compute_main_stage_loss()` 根据 `scale_mode` 选择参与 balance 的 router：

```text
fine             -> fine
fine_mid         -> fine, mid
fine_mid_coarse  -> fine, mid, coarse
```

这样 inactive scale 的 router 不会进入 importance/load balance 正则。

### 11. 训练日志增加 scale gate 指标

修改 `src/stmoe_imputer/engine.py` 和 `src/stmoe_imputer/utils/train_logger.py`。

每个 epoch 的 train/val logs 现在会包含：

```text
scale_gate_f_mean
scale_gate_m_mean
scale_gate_c_mean
scale_gate_f_std
scale_gate_m_std
scale_gate_c_std
route_gamma
effective_route_ratio
```

`TrainLogger.log_epoch()` 在原有表格行之后追加一行：

```text
metrics {...}
```

这行是 JSON，包含完整 epoch 指标，便于后续脚本解析和论文画图。

### 12. 配置更新

主配置新增：

```json
"use_scale_gate": true,
"use_reliability_gate": true
```

TaxiBJ 默认改为：

```json
"scale_mode": "fine_mid",
"lambda_cross": 0.03,
"lambda_importance_balance": 0.001,
"lambda_load_balance": 0.001
```

原因是设计文档认为 TaxiBJ 的 coarse `8x8` 可能过粗，先用 fine+mid 降低 coarse 噪声。

BikeNYC 默认保持：

```json
"scale_mode": "fine_mid_coarse",
"lambda_cross": 0.1,
"lambda_importance_balance": 0.01,
"lambda_load_balance": 0.01
```

原因是 BikeNYC 上多尺度区域趋势更明显。

新增消融配置：

- `ablation_scale_fine.json`
- `ablation_scale_fine_mid.json`
- `ablation_scale_fine_mid_coarse.json`
- `ablation_no_scale_gate.json`

用于直接跑设计文档中的 Round 1/2 scale 消融。

## 修改后的模型架构

### 总体结构

本次修改后，主分支仍保持上一轮核心范式：

```text
H_f, H_m, H_c
   ├── Reliability-aware Shared Branch -> h_shared
   └── scale_mode-aware Routed Branch  -> h_route

h_main = h_shared + sigmoid(route_gamma) * h_route_proj
x_hat_main = pred_head(h_main)
```

区别是 Shared Branch 和 Routed Branch 都开始理解 `scale_mode`。

### Shared Branch

```text
H_f, H_m, H_c
q_f, q_m, q_c
R_m, R_c
active_mask(scale_mode)
        ↓
ReliabilityAwareScaleGate
        ↓
scale_gate: [B,3]
        ↓
w_f * H_f
w_m * Up(H_m)
w_c * Up(H_c)
        ↓
Concat
        ↓
Conv3d(3D,D) + 2 x ResidualSTBlock
        ↓
z_shared
        ↓
shared_refine
        ↓
h_shared
```

### Routed Branch

`scale_mode=fine`：

```text
Z_f -> fine_refine -> h_route
```

`scale_mode=fine_mid`：

```text
Z_m -> LearnableUpsample3D -> Z_m_to_f
Z_f, Z_m_to_f -> GatedFusion2 -> h_route
```

`scale_mode=fine_mid_coarse`：

```text
Z_c -> LearnableUpsample3D -> Z_c_to_m
Z_m, Z_c_to_m -> GatedFusion2 -> Z_mc
Z_mc -> LearnableUpsample3D -> Z_mc_to_f
Z_f, Z_mc_to_f -> GatedFusion2 -> h_route
```

### 输出解释

`scale_gate` 始终是 `[B,3]`：

```text
scale_gate[:,0] -> fine 权重
scale_gate[:,1] -> mid 权重
scale_gate[:,2] -> coarse 权重
```

在不同 `scale_mode` 下：

```text
fine:
  [1,0,0]

fine_mid:
  [w_f,w_m,0]

fine_mid_coarse:
  [w_f,w_m,w_c]
```

这让后续实验可以直接观察 TaxiBJ 上 coarse 权重是否被自动压低。

## 注意事项

- 本次实现的是样本级 scale gate：`scale_gate` 形状是 `[B,3]`，不是位置级 `[B,3,T,H,W]`。
- `r_m/r_c` 已经进入 batch 和 shared gate，但如果旧 NPZ 没有 reliability，会自动用 `m_m/m_c` 近似。
- 当前 `scale_mode` 已经影响 shared branch、routed branch、cross-scale loss 和 balance loss。
- 当前仍然使用 sample-level `QualityRouter`，没有实现 patch-level router。
- `use_scale_gate=false` 时使用 active scales 均匀权重，可以作为无 gate 消融。
- 本次训练验证生成了一个 smoke 输出目录：`outputs/v2_smoke/unknown/design24_smoke_20260611_215025/`。
