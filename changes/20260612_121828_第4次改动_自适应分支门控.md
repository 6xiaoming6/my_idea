# 自适应分支门控

## 修改时间

2026-06-12 12:18:28

## 修改依据

本次修改根据 `model_designs/design_2_4.md` 的最新 v2.4 方案完成。

上一版 v2.3 已经实现：

```text
DualBranchSTImputer
  -> MultiScaleMoEBackbone
  -> GatedCrossScaleSharedExpert
  -> ProgressiveRouteFusion(scale_mode-aware)
  -> SharedRoutedResidualFusion
```

并支持：

```text
scale_mode = fine / fine_mid / fine_mid_coarse
reliability-aware scale_gate
h_main = h_shared + sigmoid(route_gamma) * h_route
```

最新 v2.4 文档指出新的核心问题：

1. TaxiBJ 上 `Routed-Only(fine_mid)` 已经优于 Full，说明 Shared Branch 可能拖累 Full。
2. 当前 Shared Branch 使用的是 pre-expert embedding `h_f/h_m/h_c`，而 Routed Branch 使用 post-expert feature `z_f/z_m/z_c`，两者特征质量不对等。
3. `route_gamma` 训练后仍然很小，固定 small residual 结构天然偏向 shared branch，不适合 TaxiBJ 这种 route-dominant 数据集。
4. TaxiBJ 和 BikeNYC 需要不同的分支融合偏置和训练超参数。

因此本次修改重点实现：

- Expert-Enhanced Shared Branch
- Adaptive Branch Gate
- residual / routed_primary / adaptive_gate 三种分支融合模式
- 参数组学习率、warmup cosine、auxiliary loss warmup、early stopping
- v2.4 推荐配置和消融配置

## 代码层面的改动

### 1. 新增 ExpertEnhancedSharedInput

在 `src/stmoe_imputer/models/fusion.py` 中新增：

```python
class ExpertEnhancedSharedInput(nn.Module)
```

作用：构造 shared branch 的输入，让 shared branch 不再只能使用 embedding 后的 pre-expert 特征，也可以使用 routed experts 输出的 post-expert 特征。

支持四种模式：

```text
pre:
  shared 输入 = h_f, h_m, h_c

post:
  shared 输入 = z_f, z_m, z_c

hybrid:
  shared 输入 = h_s + beta_s * z_s

concat_hz:
  shared 输入 = Conv1x1(concat(h_s, z_s))
```

其中 `hybrid` 是推荐主方案：

```text
h_f_shared = h_f + beta_f * z_f
h_m_shared = h_m + beta_m * z_m
h_c_shared = h_c + beta_c * z_c
```

`beta_f/beta_m/beta_c` 是可学习参数。配置中的：

```json
"shared_expert_beta_init": 0.1
```

表示有效 beta 初始值约为 `0.1`。实现时内部参数使用 `logit(beta_init)` 初始化，再通过 `sigmoid` 得到有效 beta，避免直接 `sigmoid(0.1)=0.525` 导致初始专家增强过强。

### 2. 新增 AdaptiveBranchGate

在 `src/stmoe_imputer/models/fusion.py` 中新增：

```python
class AdaptiveBranchGate(nn.Module)
```

输入：

```text
pool(h_shared): [B,D]
pool(h_route):  [B,D]
q_f:            [B,Q]
scale_gate:     [B,3]
```

拼接后经过 MLP 输出：

```text
branch_gate: [B,2]
```

含义：

```text
branch_gate[:,0] = shared branch 权重
branch_gate[:,1] = routed branch 权重
```

支持三种初始化偏置：

```text
shared_bias -> 初始更偏 shared
route_bias  -> 初始更偏 routed
balanced    -> 初始接近 0.5 / 0.5
```

这用于适配不同数据集：

```text
TaxiBJ: route_bias
BikeNYC: shared_bias
通用默认: balanced
```

### 3. 扩展 SharedRoutedResidualFusion

原来的 `SharedRoutedResidualFusion` 只支持：

```text
h_main = h_shared + sigmoid(route_gamma) * h_route_proj
```

现在支持：

```text
residual
adaptive_gate
routed_primary
```

#### residual

保留旧逻辑：

```text
h_main = h_shared + sigmoid(route_gamma) * h_route_proj
```

用于回滚或对照。

#### adaptive_gate

推荐新模式：

```text
branch_gate = AdaptiveBranchGate(h_shared, h_route_proj, q_f, scale_gate)
h_main = w_shared * h_shared + w_route * h_route_proj
```

这让模型可以根据样本观测质量和 scale gate 自适应选择 shared/routed 分支，而不是固定 shared 主导。

#### routed_primary

TaxiBJ 对照模式：

```text
h_main = h_route_proj + sigmoid(shared_gamma) * h_shared
```

用于验证 TaxiBJ 是否应以 routed branch 为主。

### 4. MultiScaleMoEBackbone 接入 expert-enhanced shared

在 `src/stmoe_imputer/models/main_branch.py` 中新增配置：

```python
shared_input_mode
shared_expert_beta_init
detach_shared_expert_input
branch_gate_init
```

forward 流程从：

```text
h_f,h_m,h_c -> GatedCrossScaleSharedExpert -> z_shared
```

改为：

```text
h_f,h_m,h_c
z_f,z_m,z_c
    -> ExpertEnhancedSharedInput
    -> h_f_shared,h_m_shared,h_c_shared
    -> GatedCrossScaleSharedExpert
    -> z_shared
```

如果：

```json
"detach_shared_expert_input": true
```

则传给 shared input adapter 的 `z_f/z_m/z_c` 会先 detach，避免 shared branch 的梯度反向干扰 routed experts。

默认：

```json
"detach_shared_expert_input": false
```

### 5. Shared-Only 也可以使用 expert-enhanced shared

之前如果 `use_routed_branch=false`，代码不会计算 `z_f/z_m/z_c`。

这会导致：

```text
Shared-Only + shared_input_mode=hybrid/post
```

实际退化为 pre-expert shared，无法验证文档中 “Shared-Only 通过 expert-enhanced shared 改善” 的目标。

本次修改后，只要：

```text
use_shared_branch = true
shared_input_mode != pre
```

即使：

```text
use_routed_branch = false
```

也会计算 routed expert features `z_f/z_m/z_c` 作为 shared branch 输入，但不会执行 `ProgressiveRouteFusion`。这样 Shared-Only 可以测试 expert-enhanced shared。

### 6. Main branch 输出新增 branch gate 和 diagnostics

`outputs["gates"]` 新增：

```python
"branch_gate": branch_gate
```

形状：

```text
[B,2]
```

`outputs["diagnostics"]` 新增：

```python
"shared_input_beta": beta
"branch_gate": branch_gate
```

顶层输出新增：

```python
"shared_input_mode"
"detach_shared_expert_input"
```

`outputs["features"]` 新增：

```python
h_f_shared
h_m_shared
h_c_shared
```

用于检查 shared branch 实际输入。

### 7. loss 新增 branch entropy 和 auxiliary warmup

在 `src/stmoe_imputer/losses.py` 中新增：

```python
categorical_entropy_loss()
```

并支持：

```json
"lambda_branch_entropy": 0.0
```

默认不启用，只预留给后续让 branch gate 更明确的实验。

`compute_main_stage_loss()` 新增参数：

```python
epoch: int | None = None
```

如果传入 epoch，则对以下 auxiliary loss 做线性 warmup：

```text
lambda_cross
lambda_importance_balance
lambda_load_balance
```

warmup 系数：

```text
warmup_factor = min(1.0, epoch / aux_loss_warmup_epochs)
```

主 loss 始终完整启用。

### 8. optimizer 参数组升级

在 `src/stmoe_imputer/engine.py` 中重写 `build_optimizer()`。

现在参数组包括：

```text
main:
  主体参数，lr = lr_main，weight_decay = weight_decay

gate:
  scale_gate / branch_gate，lr = lr_main * gate_lr_mult，weight_decay = 0

scalar:
  route_gamma / shared_gamma / shared_input beta，lr = lr_main * scalar_lr_mult，weight_decay = 0

no_decay:
  bias / norm / embedding / scale_embed，lr = lr_main，weight_decay = 0

other:
  非 main_branch 参数，lr = lr_aux，weight_decay = weight_decay
```

默认：

```json
"gate_lr_mult": 2.0,
"scalar_lr_mult": 5.0
```

这让 branch gate、scale gate 和 beta/gamma 这类门控参数能更快适配数据集。

### 9. 新增 warmup cosine scheduler

在 `src/stmoe_imputer/engine.py` 中新增：

```python
WarmupCosineLR
```

配置：

```json
"scheduler": {
  "type": "warmup_cosine",
  "warmup_epochs": 5,
  "eta_min": 1e-6
}
```

前 `warmup_epochs` 线性升学习率，之后 cosine decay。

### 10. 训练脚本新增 early stopping

在 `scripts/train.py` 中支持：

```json
"early_stopping": {
  "enabled": true,
  "monitor": "val_mae",
  "patience": 20,
  "mode": "min"
}
```

默认按 `val_mae` 监控。如果连续 `patience` 个 epoch 没有改进，会提前停止训练。

## 模型层面的改动

### 修改前 v2.3 架构

```text
H_f,H_m,H_c
  ├── GatedCrossScaleSharedExpert(H_f,H_m,H_c,q,r)
  │       -> Z_shared -> shared_refine -> H_shared
  │
  └── RoutedExperts(H_f,H_m,H_c)
          -> Z_f,Z_m,Z_c
          -> ProgressiveRouteFusion(scale_mode)
          -> H_route

H_main = H_shared + sigmoid(route_gamma) * route_proj(H_route)
X_hat = PredHead(H_main)
```

问题是：

```text
Shared Branch 使用 pre-expert H
Routed Branch 使用 post-expert Z
Full 融合天然偏 Shared
route_gamma 很小，Route 很难成为主导
```

### 修改后 v2.4 架构

```text
H_f,H_m,H_c
        ↓
RoutedExperts
        ↓
Z_f,Z_m,Z_c

Shared Branch:
  ExpertEnhancedSharedInput(H,Z)
        ↓
  H_f_shared,H_m_shared,H_c_shared
        ↓
  ReliabilityAwareScaleGate(q,r,active_scale)
        ↓
  GatedCrossScaleSharedExpert
        ↓
  Z_shared
        ↓
  shared_refine
        ↓
  H_shared

Routed Branch:
  Z_f,Z_m,Z_c
        ↓
  ProgressiveRouteFusion(scale_mode)
        ↓
  H_route
        ↓
  route_proj
        ↓
  H_route_proj

Branch Fusion:
  AdaptiveBranchGate(H_shared,H_route_proj,q_f,scale_gate)
        ↓
  branch_gate = [w_shared,w_route]
        ↓
  H_main = w_shared * H_shared + w_route * H_route_proj
        ↓
  PredHead
        ↓
  X_hat
```

这个结构使模型具备两个层面的自适应能力：

```text
scale_gate:
  决定 fine / mid / coarse 哪些尺度可信。

branch_gate:
  决定 shared / routed 哪个分支更可信。
```

预期：

```text
TaxiBJ:
  branch_gate_route_mean 应该更高，模型可以 route-dominant。

BikeNYC:
  branch_gate_shared_mean 可以更高，或 shared/route 更均衡。
```

## 注意事项

- `branch_fusion_mode="shared_plus_routed_residual"` 仍兼容旧配置，内部等价于 residual。
- `adaptive_gate` 需要 `q_f` 和 `scale_gate`，主分支已传入。
- `shared_input_mode != pre` 时，即使 `use_routed_branch=false`，也会计算 expert features 给 shared branch 使用，但不会执行 routed branch fusion。
- 本次没有实现位置级/patch级 scale gate；设计文档将其列为 P1/v2.5。
- 本次没有自动跑真实 TaxiBJ/BikeNYC 长训练，只完成了 smoke、分支模式和 1 epoch synthetic 验证。
