# Shared残差路由

## 修改时间

2026-06-10 12:09:46

## 修改依据

本次修改根据 `model_designs/design_2_3.md` 的最新方案完成第一轮核心代码改造。

该设计文档指出，当前实验中 `Shared-Only` 往往比 Full Model 更稳定，说明路由专家分支可能在 Full 模型中拖累共享分支。同时，旧版 `No Router` 消融存在语义问题：关闭 router 后虽然返回 uniform gate，但专家池仍然执行 top-k，导致实际效果不是“所有专家均匀融合”，而是“固定选择若干专家”。此外，旧版 Shared-Only/Routed-Only 消融通过 zero feature 输入同一个 fusion 模块，消融路径不够干净。

因此本次只落地文档中优先级最高的第一阶段改动：

- 修正 No Router 消融语义。
- 拆分 Shared-Only、Routed-Only、Full 的 forward 路径。
- 将 Full Model 改为 `Shared Backbone + gamma * Routed Residual`。
- 将 balance loss 从只看 soft gate usage 升级为 importance + selected load。

reliability map、patch-level router、scale_mode 等仍属于后续轮次，没有在本次一次性加入。

## 修改前的模型结构

上一次改动后，主分支结构是：

```text
x_f, m_f -> ScaleTokenEncoder -> h_f -> Router -> RoutedExperts -> z_f
x_m, m_m -> ScaleTokenEncoder -> h_m -> Router -> RoutedExperts -> z_m
x_c, m_c -> ScaleTokenEncoder -> h_c -> Router -> RoutedExperts -> z_c

h_f, h_m, h_c -> CrossScaleSharedExpert -> z_shared

z_f, z_m, z_c, z_shared
  -> ProgressiveScaleGatedFusion
  -> h_main
  -> pred_head
  -> x_hat_main
```

这个版本虽然已经把简单 concat fusion 替换成了逐级门控融合，但所有分支开关最终仍会进入同一个 `ProgressiveScaleGatedFusion`。如果 `use_routed_branch=False`，代码会把 `z_f/z_m/z_c` 置零后继续融合；如果 `use_shared_branch=False`，代码会把 `z_shared` 置零后继续融合。这样能跑，但 Shared-Only 和 Routed-Only 的消融语义不够纯粹。

旧版 No Router 路径也不干净：

```text
use_router = false
  -> gate = [1/K, 1/K, ..., 1/K]
  -> TopKRoutedExpertPool 仍执行 top-k
  -> 实际只固定选 top_k 个专家
```

这不是“无 router 的 dense all-expert 融合”，可能会导致 No Router 消融异常退化。

## 代码层面的改动

### 1. `TopKRoutedExpertPool` 支持多种 routing mode

在 `src/stmoe_imputer/models/experts.py` 中修改 `TopKRoutedExpertPool.forward()`：

```python
forward(h, gate, routing_mode="topk")
```

现在支持三种模式：

- `dense`
  - 所有专家全部参与。
  - 使用归一化后的 gate 对所有 expert outputs 加权求和。
  - 主要用于 `No Router` 消融，使 uniform gate 真正表示全专家均匀融合。

- `topk`
  - 保持原来的 hard top-k 行为。
  - 只选择 top-k 专家参与。
  - 返回 top-k 专家下标、top-k 权重和 selected mask。

- `soft_topk`
  - 先按 top-k 生成 mask。
  - 再用被 mask 的 gate 权重对 top-k 专家做向量化加权。
  - 为后续 router warm-up 或 soft top-k 实验预留入口。

返回值从旧版：

```python
z, top_indices, top_weights
```

扩展为：

```python
z, top_indices, top_weights, selected_mask
```

其中 `selected_mask` 形状是 `[B, num_experts]`，用于统计每个专家被实际选中的 load。

### 2. 新增 route-only 融合模块

在 `src/stmoe_imputer/models/fusion.py` 中新增 `ProgressiveRouteFusion`。

它只融合 routed branch 的三尺度专家输出，不再接收 `z_shared`：

```text
z_c
  -> LearnableUpsample3D
  -> z_c_to_m

z_m, z_c_to_m
  -> GatedFusion2
  -> z_mc

z_mc
  -> LearnableUpsample3D
  -> z_mc_to_f

z_f, z_mc_to_f
  -> GatedFusion2
  -> h_route
```

输出包括：

- `h_route`
- `z_c_to_m`
- `z_mc`
- `z_mc_to_f`
- `gate_16`
- `gate_32_route`

这里 `gate_32_route` 是 2 路门控，表示 fine routed 特征和 mid-coarse routed 特征之间的权重。

### 3. 新增 Shared + Routed 残差融合模块

在 `src/stmoe_imputer/models/fusion.py` 中新增 `SharedRoutedResidualFusion`。

该模块包含：

- `shared_refine`
  - `ResidualSTBlock -> ResidualSTBlock`
  - 对 `CrossScaleSharedExpert` 输出的 `z_shared` 做主干 refinement。

- `route_proj`
  - `Conv3d(dim, dim, kernel_size=1) -> ResidualSTBlock`
  - 对 routed branch 输出的 `h_route` 做投影和细化。

- `route_gamma`
  - 可学习标量参数，默认初始化为 `-4.0`。
  - 实际使用时取 `sigmoid(route_gamma)`，初始约为 `0.018`。

Full 模型的最终融合变为：

```text
h_main = h_shared + sigmoid(route_gamma) * h_route_proj
```

这样训练初始阶段模型几乎等价于 Shared-Only，只有 routed branch 学到有用补充时，`route_gamma` 才会逐渐放大 routed residual。

### 4. 主分支 forward 拆成干净路径

在 `src/stmoe_imputer/models/main_branch.py` 中，原来无条件调用：

```python
progressive_fusion(z_f, z_m, z_c, z_shared)
```

现在改为按分支开关选择路径：

```text
Shared-Only:
  z_shared -> shared_refine -> h_main

Routed-Only:
  z_f, z_m, z_c -> ProgressiveRouteFusion -> h_route -> h_main

Full:
  z_shared -> shared_refine -> h_shared
  z_f, z_m, z_c -> ProgressiveRouteFusion -> h_route
  h_shared + sigmoid(route_gamma) * route_proj(h_route) -> h_main
```

如果 `use_shared_branch=False`，不会再把 zero `z_shared` 塞进 shared+routed fusion；如果 `use_routed_branch=False`，不会再让 zero routed features 参与最终融合。

### 5. 主分支新增配置字段

`MultiScaleMoEBackbone` 新增以下配置：

```json
{
  "branch_fusion_mode": "shared_plus_routed_residual",
  "route_gamma_init": -4.0,
  "routing_mode": "topk",
  "routing_mode_when_no_router": "dense"
}
```

含义：

- `branch_fusion_mode`
  - 当前支持 `shared_plus_routed_residual`。
  - 用于明确 Full 模型采用 Shared 主干 + Routed 残差。

- `route_gamma_init`
  - 控制 routed residual 初始强度。
  - 默认 `-4.0`，经过 sigmoid 后 routed residual 初始权重大约 `0.018`。

- `routing_mode`
  - router 开启时专家池使用的融合模式。
  - 默认 `topk`。

- `routing_mode_when_no_router`
  - router 关闭时专家池使用的融合模式。
  - 默认 `dense`，用于修正 No Router 消融。

### 6. 输出字典增加可解释字段

`outputs` 新增：

```python
"selected_masks": {
    "fine": selected_f,
    "mid": selected_m,
    "coarse": selected_c,
}
```

用于统计专家实际 load。

`outputs["features"]` 新增或保留：

- `h_f`, `h_m`, `h_c`
- `z_f`, `z_m`, `z_c`
- `z_shared`
- `h_shared`
- `h_route`
- `h_route_proj`
- `h_main`
- `z_c_to_m`
- `z_mc`
- `z_mc_to_f`

`outputs["gates"]` 中 routed fusion 门控改为：

- `route_fusion_16`: `[B, 2, T, H/2, W/2]`
- `route_fusion_32`: `[B, 2, T, H, W]`

同时顶层新增：

- `routing_mode`
- `branch_mode`
- `route_gamma`

这些字段可以直接用于实验日志和论文分析。

### 7. balance loss 升级

在 `src/stmoe_imputer/losses.py` 中新增 `moe_balance_loss()`。

旧版只看 soft gate 平均使用率：

```text
importance = mean(gate)
loss = ||importance - 1/K||^2
```

新版同时计算：

```text
importance: 专家获得的 soft gate 权重均值
load:       专家被 selected_mask 实际选中的比例
```

最终 loss 拆成：

- `l_importance_balance`
- `l_load_balance`
- `l_balance = l_importance_balance + l_load_balance`

如果是 `use_router=false` 且 `routing_mode=dense`，balance loss 自动置 0，因为 No Router dense 消融本来就是人为均匀融合，不应该再被 MoE balance 正则影响。

同时新增 `fusion_entropy_loss()`，但默认 `lambda_fusion_entropy=0.0`，本次不启用，只为后续门控熵正则预留接口。

### 8. 配置文件更新

在 `default.json`、`taxibj.json`、`bikenyc.json`、`smoke.json` 中补充：

```json
"branch_fusion_mode": "shared_plus_routed_residual",
"route_gamma_init": -4.0,
"routing_mode": "topk",
"routing_mode_when_no_router": "dense",
"router_granularity": "sample",
"use_reliability": false,
"scale_mode": "fine_mid_coarse"
```

其中 `router_granularity`、`use_reliability`、`scale_mode` 目前是显式记录后续设计方向的配置字段，本次代码仍使用 sample-level router，且还没有接入 reliability 或 scale_mode 分支逻辑。

loss 配置新增：

```json
"lambda_importance_balance": 0.01,
"lambda_load_balance": 0.01,
"lambda_fusion_entropy": 0.0
```

旧的 `lambda_balance` 保留为兼容字段。

新增消融配置：

- `ablation_no_router_dense.json`
  - `use_router=false`
  - `routing_mode_when_no_router=dense`
  - 用于新的干净 No Router 消融。

- `ablation_no_router_topk.json`
  - `use_router=false`
  - `routing_mode_when_no_router=topk`
  - 用于复现旧 No Router 可能固定选 top-k 的行为，方便对照。

- `ablation_shared_clean.json`
  - `use_routed_branch=false`
  - `use_shared_branch=true`
  - clean Shared-Only 路径。

- `ablation_shared_plus_routed_residual.json`
  - `use_routed_branch=true`
  - `use_shared_branch=true`
  - `branch_fusion_mode=shared_plus_routed_residual`
  - Full 新结构消融。

旧的 `ablation_no_router.json` 也改成默认 dense 语义。

## 修改后的模型架构

### Shared-Only

```text
x_f, m_f -> ScaleTokenEncoder -> h_f
x_m, m_m -> ScaleTokenEncoder -> h_m
x_c, m_c -> ScaleTokenEncoder -> h_c

h_f, h_m, h_c
  -> CrossScaleSharedExpert
  -> z_shared
  -> shared_refine
  -> h_main
  -> pred_head
  -> x_hat_main
```

Shared-Only 现在就是纯共享分支主干，不再让 zero routed features 进入最终融合。

### Routed-Only

```text
x_f, m_f -> ScaleTokenEncoder -> h_f -> Router -> RoutedExperts -> z_f
x_m, m_m -> ScaleTokenEncoder -> h_m -> Router -> RoutedExperts -> z_m
x_c, m_c -> ScaleTokenEncoder -> h_c -> Router -> RoutedExperts -> z_c

z_c -> LearnableUpsample3D -> z_c_to_m
z_m, z_c_to_m -> GatedFusion2 -> z_mc
z_mc -> LearnableUpsample3D -> z_mc_to_f
z_f, z_mc_to_f -> GatedFusion2 -> h_route

h_route -> pred_head -> x_hat_main
```

Routed-Only 现在只看 routed branch，不再混入 zero shared branch。

### Full Model

```text
Shared Backbone:
  h_f, h_m, h_c
    -> CrossScaleSharedExpert
    -> z_shared
    -> shared_refine
    -> h_shared

Routed Residual:
  h_f, h_m, h_c
    -> Router
    -> TopKRoutedExpertPool
    -> z_f, z_m, z_c
    -> ProgressiveRouteFusion
    -> h_route
    -> route_proj
    -> h_route_proj

Final:
  gamma = sigmoid(route_gamma)
  h_main = h_shared + gamma * h_route_proj
  x_hat_main = pred_head(h_main)
```

初始化时 `route_gamma_init=-4.0`，所以：

```text
gamma = sigmoid(-4.0) ≈ 0.018
```

因此 Full Model 一开始接近 Shared-Only。训练过程中如果 routed branch 有帮助，模型可以逐步增大 gamma；如果 routed branch 没用，gamma 可以保持较小，从而降低 routed branch 拖累主干的风险。

## No Router 消融语义变化

修改前：

```text
use_router=false
  -> uniform gate
  -> top-k
  -> 固定选部分专家
```

修改后默认：

```text
use_router=false
  -> uniform gate
  -> dense all experts
  -> 所有专家等权参与
```

这使 No Router 消融真正表示“没有样本级路由决策，所有专家均匀融合”。

如果需要复现旧行为，可以使用：

```text
configs/ablation_no_router_topk.json
```

## 注意事项

- `ProgressiveScaleGatedFusion` 仍保留在 `fusion.py` 中并继续导出，但当前 `MultiScaleMoEBackbone` 默认不再使用它作为 Full 的最终融合器。
- 旧输出中的 `fusion_16/fusion_32` 已改为 route-only 语义下的 `route_fusion_16/route_fusion_32`。
- `route_fusion_32` 现在是 2 路门控，不再是包含 shared 的 3 路门控；shared 与 routed 的最终权重由标量 `route_gamma` 控制。
- 当前还没有接入 reliability map，配置中的 `use_reliability=false` 只是明确记录本轮未启用该特性。
- 当前还没有实现 patch-level router，配置中的 `router_granularity=sample` 表示仍使用原来的样本级 `QualityRouter`。
- 当前还没有实现 `scale_mode` 分支逻辑，`scale_mode=fine_mid_coarse` 只是为后续多尺度数量消融预留配置字段。
